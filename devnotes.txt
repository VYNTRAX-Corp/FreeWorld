Day 4 of building my own FPS game from scratch 🎮

Enemies now support REAL human skins 🧑‍🤝‍🧑

Been using procedural primitive bodies since day 2 —
cubes, cylinders, spheres stitched together.
Today I built the full pipeline to swap those out for actual CC0 humanoid models.

How it works under the hood:
→ Drop any CC0 humanoid FBX into Assets/Characters/
→ Hit "Configure Models" in the new editor tool
→ It auto-sets Humanoid rig, bakes the avatar, generates an AnimatorController
→ Assign the prefab to the enemy in the Inspector — done

The enemy AI doesn't care which body it's running on:
Both paths (procedural primitives OR real FBX) return the same
EnemyBodyParts struct with bone Transform references.
EnemyModelAnimator drives the Animator via Speed, IsAttacking,
IsReloading, IsDead parameters — the AI logic didn't change at all.

The gun automatically attaches to the right forearm bone either way. 🔫

The AnimatorController comes pre-wired:
Idle → Walk → Run → Aim → Reload → Death
All transitions, all parameters — just drag your animation clips in.

Best CC0 character sources I found:
🎭 Quaternius (quaternius.itch.io) — low poly humanoids, CC0, includes animations
🎭 Kenney — toon characters pack, CC0, clean topology
🎭 OpenGameArt — various CC0 base meshes

The whole system is a clean fallback chain:
Real model → procedural if no model assigned
No code changes needed in the AI when you switch

Everything is still 100% code, zero bought assets

Built with Unity + GitHub Copilot 🤖

Enemies finally look like actual soldiers 🪖

Built a full procedural humanoid from scratch — head, torso,

Then gave them real guns.
Not just "shoot a raycast" — a full shooting system:

But then I had to fix some nasty bugs to make them actually work —
enemies weren't respawning, the timer was frozen, kill counter stuck at 0...

Turns out ONE component sharing a GameObject with another
was silently destroying the entire game manager every session 💀
